# v0.6.0

## Branch Switcher

Added a branch picker to the Git Status panel. Click the "..." button next to the branch name to open a popup with:

- **Filter textbox** — type to filter existing local branches with autocomplete
- **Branch list** — shows all matching branches with a checkmark on the current branch; double-click or press Enter to switch
- **Create new branch** — when the typed name doesn't match any existing branch, a "+" button appears to create and switch to a new branch

Keyboard shortcuts: Down arrow moves to the list, Enter selects/creates, Escape closes the popup.

## Open Logs Folder

Added **Settings > Open Logs Folder** menu item. Clicking it opens Windows Explorer to the logs directory, making it easy to find and share log files.

## Markdown Heading Sizes

Fixed markdown headings (`#`, `##`, `###`, `####`) all rendering at the same font size in both the file preview panel and AI chat markdown view. Headings now render at distinct sizes matching their level.

## Documentation Overhaul

Reorganized documentation into dedicated pages by topic:

- **[Features](../features.md)** — feature overview with links to detailed docs
- **[Workspace](../workspace.md)** — saving, loading, themes, and settings
- **[Projects](../projects.md)** — tabs, project settings, and panel layouts
- **[Project Features](../project-features.md)** — File Explorer, Git Status, and File Preview
- **[AI Chat](../ai-chat.md)** — sessions, message types, and troubleshooting

INSTALL.md slimmed down to installation and quick start, with links to the new docs.

## AI Chat Scroll Fix

Fixed inconsistent mouse wheel scrolling in the AI chat panel. Child elements (selectable text blocks, markdown viewers) no longer steal wheel events — the chat always scrolls smoothly.

## File Explorer Refresh Button

Added a manual **Refresh** button (circular arrow icon) to the File Explorer toolbar. Useful when automatic file watching misses changes.

## Process Exit Crash Fix

Fixed a crash caused by a .NET race condition when a Claude Code subprocess exits. The `Process.Exited` event could fire before the OS handle was fully ready, causing `ExitCode` access to throw `InvalidOperationException` on a background thread — which was unhandled and crashed the app.

## Inactivity Warning Improvements

Improved the "Claude may be stuck" warning that appears after prolonged inactivity:

- **Wait longer** button — dismisses the warning and resets the inactivity timer so Claude can keep working
- **Elapsed time** — shows how long since the last message was sent, updated every second
