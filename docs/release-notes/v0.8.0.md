# v0.8.0

## Fix Token and Cost Tracking Inflation

- Fixed a bug where session token counts and cost were massively inflated over time
- Claude Code reports cumulative totals in each result message, but AgentDock was summing them as incremental deltas — causing quadratic growth over many interactions
- Session stats now correctly replace cumulative values and compute per-interaction deltas for display

## Sound Notifications

- Plays Windows **device connect** sound when a session starts
- Plays Windows **message nudge** sound when an agent finishes and is waiting for user input (diamond turns green)
- Plays Windows **device disconnect** sound when a session ends
- Uses the user's configured Windows sound scheme — respects mute and custom sounds

## Shared Markdown Rendering

- Extracted shared MarkdownHelper so AI chat and file preview use the same rendering pipeline
- AI chat now gets the same bold/italic link fixups and code block theming as file preview

## Slash Command Autocomplete

- Type `/` in the chat input to see a dropdown of all available Claude Code slash commands with descriptions
- Up/Down arrows to navigate, Enter or Tab to insert, Escape to dismiss
- Filters as you type — `/com` narrows to `/compact` etc.
- Includes Claude Code 2.1.x commands from the official docs plus Agent Dock local commands (`/clear`, `/stop`, `/logs`)
- Primary command names only; aliases (e.g. `/settings` → `/config`) are shown in the description

## Model Name in AI Chat Panel Title

- AI chat panel title now shows the active Claude model (e.g. `Sonnet 4.5 — $0.0123 · 1.2k tokens`) once the session reports it
- Raw model ids like `claude-sonnet-4-5-20250929` are shortened to `Sonnet 4.5` (family + version, trailing date dropped)
- Falls back to `AI Chat` before the session starts

## Per-Project Sound Settings

- Project Settings dialog now has a **Sounds** section with a `Configure...` button
- Opens an inline panel with three checkboxes — one per sound event:
  - Session start (device connect)
  - Agent waiting for input (message nudge)
  - Session end (device disconnect)
- Summary label shows `All enabled`, `All disabled`, or `N of 3 enabled` so you can tell at a glance without expanding
- Defaults to all on, preserving existing behaviour for upgraders
- Persisted per-project in `.agentdock/settings.json` (`soundOnSessionStart`, `soundOnAgentWaiting`, `soundOnSessionEnd`)

## Tab Diamond: Flash on Completion

- When a background tab's Claude session transitions from Working (or WaitingForPermission) to Idle, the green diamond in that tab now flashes gently to draw the user's attention to the unread completion
- Flashing stops the moment the user switches to that tab — the diamond becomes solid green
- Already-active tabs never flash (the user can see the completion in the chat panel)
- Uses a subtle 700ms opacity pulse (1.0 → 0.55), slower and less aggressive than the Working pulse

## Git Status: Handle Non-Git Folders

- Opening a project folder that isn't a git repository now correctly shows "Not a git repository" in the Git Status panel and keeps showing it
- Previously, the "Not a git repository" message would briefly appear on load but get replaced with "No changes" whenever the theme changed or the AI session went idle, because `RefreshStatus` was rebuilding the panel unconditionally
- No `git status` subprocess calls, file-system watchers, or debounce/polling timers are started for non-git folders

## Custom Project Name

- Project Settings dialog now has a **Project Name** field at the top
- Leave it blank to keep using the folder name (the default); type a name to override
- Custom name appears in the project tab, File Explorer panel title, File Explorer window title, and the main window title
- Persisted in `.agentdock/settings.json` as the new `name` field
- Other settings (description, todo items, description font size) are no longer wiped when OK'ing the Project Settings dialog — previously unrelated fields were overwritten on save

## Plan Usage Indicator in Title Bar

- Shows the current 5-hour session utilization and time-to-reset in the title bar (e.g. `5h: 24% · resets in 2h 14m`)
- Click for a popup with the full breakdown: 5-hour window, 7-day weekly window, per-model (Opus/Sonnet) usage where available, and extra-usage credits
- Polls the usage endpoint on demand: only fetches when a Claude Code session has actually consumed usage since the last fetch, otherwise just re-renders the countdown text every 90 seconds. This avoids HTTP 429s when AgentDock sits idle.
- Manual refresh button in the popup forces an immediate fetch regardless
- Data sourced from the same `GET /api/oauth/usage` endpoint Claude Code's `/status` tab uses (no inference tokens consumed)
- Shows sign-in / offline / expired states when the OAuth token from `~/.claude/.credentials.json` isn't usable
